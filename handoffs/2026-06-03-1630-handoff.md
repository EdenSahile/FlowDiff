### `src/lib/ediUtils.ts` — refonte template ORDERS
- `UNOA:1` → `UNOC:3` dans UNB (charset pour accents français)
- GLN placeholders lisibles : `ClientGLN` et `DiffuseurGLN` (au lieu de `301234XXXXXXX` / `GLN-DIFFUSEUR`)
- `UNH+{docRef}+ORDERS:D:96A:UN:EAN008'` — réf. message = documentRef (pas `+1`)
- `CUX+2:EUR:9'` — qualificateur 9 = devise commande (était 4 = devise facturation)
- `RFF+LI` (line item) pour réf. par ligne, au lieu de `RFF+CR` (customer ref générique)
- `RFF+API:clientCode'` dans le header (N° compte libraire chez le diffuseur)
- `UNA:+.? '` ajouté en tête du message
- `IMD+L+009/010/011` — auteur (complet / nom / prénom) via helper `parseAuthorImd()`
- `IMD+L+050` — titre avec split automatique à 35 chars via helper `titleToImd050()`
- `IMD+L+109` — éditeur
- `IMD+L+170` — année de publication
- `IMD+L+180+:::Livre'` — type de média (existait, conservé)
- `CNT+1:{nbLignes}'` + `CNT+2:{qtyTotale}'` après `UNS+S'`
- `UNT+{count}+{docRef}'` — réf. message = documentRef (était `+1`)
- Helpers ajoutés : `escEdifact()` (module-level), `parseAuthorImd()`, `titleToImd050()`
- Interfaces enrichies : `ORDERSLine` (+`authors?`, `publisher?`, `publishYear?`) · `ORDERSPayload` (+`clientCode?`)

## Contexte métier important (mémorisé)
- **Un seul diffuseur** dans FlowDiff (l'utilisateur lui-même). Ne jamais afficher "Diffuseur 1/2/3/4" en production. `DiffuseurGLN` constant est correct.
- `RFF+LI` = référence par ligne (line item) · `RFF+CR` = référence globale commande (header) · `RFF+API` = code compte libraire
