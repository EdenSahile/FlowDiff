## Blocages / échecs

### Diagnostic clé établi via Playwright
- L'app charge **67 livres depuis Supabase** (table `livres`), pas les mockBooks — `src/services/books.ts` priorise Supabase
- Les ISBNs Supabase sont complètement différents des ISBNs mockBooks (ex: `9782070408504` Le Petit Prince)
- Résultat : le `DIRECT_COVER_URLS` (39 ISBNs mockBooks) **n'est jamais utilisé** pour les livres affichés
- La clé `VITE_GOOGLE_BOOKS_API_KEY` **ne fonctionne pas depuis le navigateur** (`localhost`) — restriction de domaine probable dans Google Console — les appels API depuis le browser retournent `hasItems: false`
- Les livres Supabase sans cover tombent en Open Library → ceux qu'Open Library ne connaît pas affichent "image not available" (le placeholder OL est > 10px donc il passe le check `naturalWidth < 10`)
- Sur la page Fonds : 19 images Google Books (Supabase books avec cover URL déjà stockée en DB) + 43 Open Library (dont un nombre inconnu "image not available")

### Approches qui ne règlent pas le vrai problème
- Corriger les ISBNs de mockBooks : inutile si Supabase override tout
- `DIRECT_COVER_URLS` dans BookCover.tsx : inutile pour les ISBNs Supabase
